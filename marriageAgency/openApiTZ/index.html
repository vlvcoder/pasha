2. Интерфейс пользовательских постов /posts

Управление постами, привязанными к пользователю.

GET /users/{userId}/posts query: postDate

Запрос списка постов пользователя

параметр тип / описание
userId path, обязательный, integer
postDate query, необязательный, string (date)
Тело запроса отсутствует


✅ Возврат (200)
[
  {
    "id": 1234,
    "userId": 314,
    "date": "2001-03-23",
    "time": "15:55",
    "text": "Hi all! I am fine",
    "smileColor": "green"
  },
  {
    "id": 1235,
    "userId": 314,
    "date": "2001-04-23",
    "time": "03:00",
    "text": "I want to get married!"
    "smileColor": "green"
  }
]
smileColor допустимые значения:
none, red, green, yellow
Используйте enum

❌ Возврат по ошибке
{
    "message": "Ошибка выполнения операции",
    "details": "Видимо, что-то случилось",
}

Валидация smileColor: поле принимает только строки из фиксированного набора:
  • none
  • red
  • green
  • yellow

Общие требования к OpenAPI-спецификации

  • Формат описания — YAML или JSON (ссылочная модель ma-openapi.json).
  • Корректно описать все path- и query-параметры, схемы ответов (200, 201, default).
  • Для smileColor использовать enum (none, red, green, yellow).
  • Модели User, Post вынести в components/schemas.

Пример выполнен в качестве образца в ma-openapi.json — структура должна быть расширена, но сохранена стилистика.
